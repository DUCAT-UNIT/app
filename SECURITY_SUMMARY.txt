HIGH SEVERITY ISSUES (Fix Before Testnet)
================================================================================

1. INSECURE HKDF IMPLEMENTATION
   File: /services/passkeyService.js lines 108-124
   Issue: Not RFC 5869 compliant, uses string concat instead of HMAC
   Fix: Use react-native-quick-crypto for proper HMAC-based HKDF

2. RACE CONDITION IN PIN CHANGE
   File: /services/passkeyService.js lines 862-931
   Issue: Not atomic - could leave wallet in inconsistent state
   Fix: Add semaphore flag to prevent concurrent PIN operations

3. UNSANITIZED ERRORS IN SIGNING
   File: /services/transactionSigningService.js line 198
   Issue: Could expose key material in error messages
   Fix: Sanitize with regex patterns for sensitive data

4. PASSKEY CHALLENGE REPLAY VULNERABILITY
   File: /services/passkeyService.js
   Issue: No timestamp validation on WebAuthn challenges
   Fix: Add timestamp to challenge, validate freshness

5. NO BIOMETRIC RATE LIMITING
   File: All auth flows
   Issue: Unlike PIN (10 attempts), biometric has unlimited retries
   Fix: Add biometric attempt tracking with lockout

6. TRANSACTION FINALIZATION ERROR SWALLOWING
   File: /services/transactionSigningService.js lines 156-170
   Issue: Catches all errors without logging which path taken
   Fix: Log finalization path, only catch specific errors

7. SENTRY NOT SANITIZING MNEMONIC/KEYS
   File: /App.js lines 38-50
   Issue: Stack traces and variables might contain sensitive data
   Fix: Implement sanitization patterns in Sentry beforeSend hook

8. NO TRANSACTION MALLEABILITY CHECK
   File: /services/transactionSigningService.js
   Issue: Could broadcast malleated transactions with different TXIDs
   Fix: Verify TXID after signing matches expected value
