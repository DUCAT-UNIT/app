4. CRITICAL ISSUES (Priority 1)
================================================================================

ISSUE #1: authService.js Oversized (406 lines)
─────────────────────────────────────────────
Current: Single 406-line service mixing:
  - Biometric authentication (105 lines)
  - PIN setup/verification (120 lines)
  - Session management (95 lines)
  - Secure storage (86 lines)

Recommendation: Split into 3 services
  - authService.js (200 lines) - Core auth/session
  - pinService.js (150 lines) - PIN setup/verification
  - biometricService.js (100 lines) - Biometric operations

Impact: Cleaner separation, easier testing, better maintainability
Risk: Low (good test coverage present)
Effort: 4-6 hours


ISSUE #2: PendingTransactionsContext Oversized (343 lines)
──────────────────────────────────────────────────────────
Current: Context mixing state with polling logic
  - pendingTransactions array (state)
  - Polling loop logic (useEffect heavy)
  - Background task integration
  - Transaction tracking

Recommendation: Extract polling to custom hook
  - usePendingTransactionPolling.js (150 lines)
  - Reduce context to 200 lines

Impact: Cleaner context, reusable polling hook
Risk: Low
Effort: 3-4 hours


ISSUE #3: useSettings Hook Oversized (295 lines)
────────────────────────────────────────────────
Current: Single 295-line hook handling:
  - Display preferences/persistence
  - Settings navigation
  - Display preference logic
  - Multiple unrelated domains

Recommendation: Split into 4 focused hooks
  - useDisplaySettings.js (100 lines)
  - useSecuritySettings.js (80 lines)
  - useGeneralSettings.js (70 lines)
  - useSettingsNavigation.js (65 lines)

Impact: Reusability, single responsibility
Risk: Low
Effort: 5-6 hours
